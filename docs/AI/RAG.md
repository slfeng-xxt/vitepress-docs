# RAG - 检索增强生成

:::tip
先从资料库里**检索**相关内容

再基于这些内容来**生成**答案
:::

## 1. 总体介绍

### 1.1 使用场景

例如：公司内部需要一个本公司产品解决方案的人工智能助手。

### 1.2 大致流程

当使用大模型（chatgpt/claude/...）但是这些大模型没有本公司的产品介绍和知识内容，那么就需要将产品的使用手册等相关文档一股脑塞给大模型，这样token消耗巨大，不合适；或者大模型接受不了上下文，会降低模型等性能。那么需要RAG来解决。

```text
如何解决呢？

- 1.将用户手册内容分成不同的片段
- 2.将查询到相关的手册内容的部分片段塞给大模型，这样就只取有用的信息给大模型，让大模型来处理问题
```

### 1.3 基本流程

- 准备「提问前」： 分片/索引
- 回答「提问后」：召回/重排/生成

## 2. 工作原理

### 2.1 分片

- 将文档切分为多个片段，分片方式：按字数/按段落/按章节/按页码

### 2.2 索引

:::tip
向量：有大小有方向的量,数组方式表达

Embedding:把文本转为向量的过程

向量数据库：用于存储文本和查询向量的数据库
:::

- 通过 **「Embedding」**将片段文本转化为 **「向量」**
- 将片段文本和片段向量存入 **「向量数据库」**

```text
大致流程如下：

文本内容 ---> Embedding  ---> 向量[1,3,4,5, ....] + 「原始文本片段」  ---> 存入向量数据库

查询到相似的向量后，将相似向量的文本返回
```

### 2.3 召回

- 搜索和用户问题相关的片段的过程（即索引过程结束后返回的与用户问题相关的「原始文本片段」。例如，返回10个片段段文本）
- 原理需要计算向量相似度

:::tip
向量相似度计算方式：

1. 余弦相似度：两个向量之间的夹角的cos值，越小相似度越高

2. 欧式距离：两个向量指向终点之间的距离，越近相似度越高

3. 点积：代数方式计算，不仅考虑夹角还考虑距离，a向量投影到b向量上的距离✖️b向量，根据值来判断相似度，越大越相似

:::

### 2.4 重排（重新排序）

- 从召回中返回的10个片段，再重新找3个片段

### 2.5 生成

- 拿到用户问题和查到的资料片段，丢给大模型 ， 大模型最后生成回复

### 2.6 召回和重排相似度的区别

| 区别内容 | 召回（简历筛选） | 重排（面试） |
| --- | --- | --- |
| 方法 | 向量相似度 | cross-encorder |
| 特点 | 成本低/耗时短/准确率低 | 成本高/耗时长/准确高 |
| 适合场景 | 初步筛选 | 精挑细选 |

## 3. 全链路

### 3.1 提问前链路

- 将公司手册资料分片
- 分片列表
- 转化为向量
- 将向量和对应的原始文本一起存入向量数据库

### 3.2 提问后链路

- 将用户的问题转化为向量
- 进入向量数据库筛选一批相似度高的向量和原始文本
- 将查到的这一批向量继续计算一遍，查到3个更高 相似度的原始文本
- 将用户问题和更高的相似度的原始文本一起丢给大模型处理
- 最后大模型返回结果
